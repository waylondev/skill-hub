# Excel 物流回填助手

## 触发条件

当用户说出以下类似内容时触发此SKILL：
- "帮我回填物流"
- "把物流单号匹配到原单"
- "这两个Excel帮我匹配一下"
- "回填快递单号"
- 用户提供原单和回单文件，要求匹配物流信息

## 功能概述

自动完成以下操作：
1. 扫描目录，识别所有回单文件（邮政、中通等）
2. 智能识别原单和回单的列语义（不依赖固定列名）
3. 通过**姓名+电话+地址**多列组合匹配，依次用每个回单回填原单
4. 输出回填结果文件和匹配统计报告

## 核心匹配逻辑

**回单中没有订单号！使用多列组合匹配。**

| 优先级 | 匹配键 | 说明 |
|--------|--------|------|
| 1 | 姓名+电话+地址 | 最精确，三列完全一致 |
| 2 | 电话+地址 | 电话相同且地址包含关系 |
| 3 | 仅电话 | 兜底匹配，仅当电话在回单中唯一时使用 |

**支持多回单匹配**：依次用每个回单文件匹配，已匹配的订单跳过。

## 智能列识别规则

**不要硬编码列名！根据列名关键词+数据特征识别。**

| 列类型 | 列名关键词 | 数据特征验证 |
|--------|-----------|-------------|
| 电话 | 电话、手机、联系方式、号码、phone、tel | 11位数字，1开头，正则`^1[3-9]\d{9}$` |
| 姓名 | 姓名、收件人、收货人、购买人、客户、name | 2-8字符，主要是中文 |
| 地址 | 地址、省、市、区、县、镇、路、address | 包含2个以上地址关键词，长度>10 |
| 物流单号 | 包裹号、运单编号、运单号、物流单号、快递单号 | 8-20位，纯数字或字母数字 |
| 物流公司 | 物流公司、快递公司、承运商 | 包含"中通""邮政""顺丰"等 |

**识别顺序**: 先看列名关键词 → 再看数据样本验证 → 根据特征判断

## 执行步骤

### Step 1: 确认文件

1. 确认用户提供的原单文件路径
2. 扫描原单所在目录，找出所有 `.xls` / `.xlsx` 文件
3. 过滤掉原单本身，保留回单文件（表头通常含"编号"+"包裹号/运单号"）
4. 向用户确认识别到的文件是否正确

### Step 2: 检查环境

检查Python环境是否需要安装依赖：
```bash
uv pip install pandas openpyxl xlrd
```

### Step 3: 分析原单

1. 读取原单Excel（xlsx用pandas，xls用xlrd）
2. 智能识别三个关键列：姓名列、电话列、地址列
3. 找到或新增物流单号列和物流公司列

### Step 4: 分析回单

对每个回单文件：
1. 读取回单数据和表头
2. 从表头提取物流公司名称（含"中通"=中通，含"邮政/EMS"=邮政）
3. 智能识别四个关键列：姓名列、电话列、地址列、物流单号列
4. 构建电话→物流单号映射

### Step 5: 依次匹配回填

对每个回单依次执行：
1. 遍历原单每条记录，跳过已有物流单号的订单
2. 按优先级匹配：
   - 优先级1: 姓名+电话+地址 三列一致
   - 优先级2: 电话+地址 一致（地址包含判断）
   - 优先级3: 仅电话一致（且该电话在回单中唯一）
3. 匹配成功则回填物流单号和物流公司

### Step 6: 输出结果

向用户展示：
1. 每个回单的匹配统计（回单名称、物流公司、匹配数）
2. 最终汇总（总订单数、匹配成功、未匹配、匹配率）
3. 未匹配订单示例（前3-5个，含姓名、电话、订单号）
4. 输出文件路径

保存结果到新Excel文件。

## 异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| 文件不存在 | 提示用户确认文件路径 |
| 文件格式错误 | 提示用户检查文件格式，尝试用不同方式读取 |
| 无法识别关键列 | 打印所有列名，请用户确认哪列是电话/姓名/地址 |
| Python环境缺少依赖 | 自动安装 pandas、openpyxl、xlrd |
| 匹配率<50% | 提醒用户检查是否缺少某些物流公司的回单文件 |
| 回单表头格式异常 | 根据数据特征而非列名识别列用途 |
